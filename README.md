##Тестовое задание - приложение для просмотра документов и добавления аннотаций на Angular.

## Установка и запуск
-npm install
-ng serve или npm start


Плюсы реализации:
-Чёткое разделение ответственности между компонентами
-Изолированный API-слой, легко заменить mock на реальный backend
-Реализация drag-and-drop без сторонних библиотек
-Использование standalone компонентов
-Использование TypeScript, строгие модели данных

Минусы/Ограничения реализации:

-Координаты аннотаций хранятся в пикселях и масштабируются вместе с документом
-Нет ограничений перемещения аннотаций по границам страницы
-Состояние документа не сохраняется между перезагрузками
-Отсутствуют loading и error states
-Отсутствие тестов

Известные проблемы:

-При изменении zoom координаты аннотаций могут визуально смещаться
-Аннотации можно переместить за пределы страницы
-Ошибки загрузки документа не обрабатываются

Возможные способы улучшения:

-Хранить координаты аннотаций в процентах относительно страницы
-Ограничить перемещение аннотаций размерами страницы, добавить валидацию границ
-Добавить сохранение состояния (localStorage или backend)
-Реализовать отмену действий/редактирование аннотаций
-Добавить обработку ошибок и индикаторы загрузки
-Подключить реальный backend API
-Добавить unit и e2e тесты
-Вынесение текстов UI в constants
-Вместо input можно использовать MatDialog с формой для ввода текста аннотации
-Улучшить стили, сделать более красивое отображение
